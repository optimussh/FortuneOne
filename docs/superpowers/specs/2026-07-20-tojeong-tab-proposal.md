## 1. 어디에 넣는 게 좋은가?

### 권장: `/me` 상세 사주 **탭 추가**

이미 있는 탭:

| 기존 탭 | 역할 |
|---------|------|
| 오늘의 운세 | 일운 (매일 변함) |
| 2026 신년 | 한 해 테마 (고정) |
| 오행 사주 | 기질·구조 (고정) |
| 인생풀이 | 초중말·평생 (고정) |

**추가 탭 제안:**

| 새 탭 | 이름(UI) | 역할 |
|-------|----------|------|
| **tojeong** | **명품 연간(토정)** | 예시 형식의 **연간 종합 + 월별 + 영역운** |

### 왜 `/me` 탭인가?

1. **입력 프로필(이름·양력·시진)** 을 이미 쓰는 화면이라, 예시의  
   `강석훈 / 1990.08.27 / 辰(07:30–09:29) / 사주명식` 헤더와 자연스럽게 붙음.
2. **신년 탭과 역할 분리** 가능  
   - 신년 = 테마·상하반기·로드맵 (지금 구조)  
   - 토정 = 문장형 종합운 + **12개월 월별** + 연애/직장/건강/소망/대인/처세 + 숫자·색
3. 허브(`/hub`)는 “매일 짧게” 유지. 토정은 **긴 리포트**라 허브에 넣으면 무거움.
4. 메뉴: 허브에 “2026 토정 보기” CTA → `/me?tab=tojeong` 링크만 추가.

### 비권장

- **독립 URL만** (`/tojeong`) — 가능하지만 프로필 선택·명식 UI를 또 만들어야 함.  
  → 1차는 `/me` 탭, 나중에 `/tojeong` 딥링크 허용.

## 2. 예시 형식 → 우리 앱 섹션 매핑 (중복 제거)

예시에서 **겹치는 것**을 빼고 한 번만 배치:

| 예시 블록 | 반영 위치 | 비고 |
|-----------|-----------|------|
| 이름·양력·시진 한 줄 | 리포트 **헤더** (모든 탭 공통) | 프로필에서 가져옴 |
| 사주명식 천간/지지 + 십성 | **오행 탭 상단** 또는 토정 탭 상단 **명식 카드 1회** | 중복 방지: 토정 탭에만 풀 표, 오행은 요약 |
| 오행 木火土金水 바 | **오행 탭** (기존 유지) | 토정에서는 한 줄 요약만 |
| 올해의 종합운 (장문) | **토정 탭 · 종합운** | 신년 탭과 문체 다르게 (속담·비유형) |
| 월별운 1–12월 | **토정 탭 · 월별** | 신년에는 월별 없음 → **여기가 핵심 차별** |
| 연애/직장/건강/소망/대인/처세 | **토정 탭 · 영역운** | 기존 주제 운세(`/topics`)와 겹치면 토정은 “연간”, topics는 “오늘” |
| 행운/불운 숫자·색 | **토정 탭 · 부적 정보** | 기존 일운 lucky와 분리 (연간 고정) |
| 예시 하단 중복 월별 | **제거** | 월별은 12칸 한 세트만 |

### 제안 탭 구조 (토정 화면 내부)

```
[ 헤더: 이름 · 양력 · 시진 · 명식 미니표 ]
[ 서브탭 또는 아코디언 ]
  1. 종합운
  2. 월별운 (12개월 카드)
  3. 영역운 (연애·직장·건강·소망·대인·처세)
  4. 행운 숫자·색
```

## 3. 기존 탭과 역할 정리 (사용자 혼란 방지)

| 탭 | 한 줄 | 고정? |
|----|--------|-------|
| 오늘 | 오늘 하루 | 날짜마다 변경 |
| 신년 | 올해 테마·상하반기·로드맵 (분석형) | 연도+사주 고정 |
| **토정** | **올해 이야기형 + 월별 12 + 영역운** | 연도+사주 고정 |
| 오행 | 나는 어떤 구조인가 | 사주 고정 |
| 인생풀이 | 초중말·평생 | 사주 고정 |

**신년 vs 토정**  
- 신년: 계획·로드맵 톤  
- 토정: 전통 문장·월별·속담 톤 (예시 느낌)  
둘 다 “2026”이어도 **콘텐츠 모듈이 다름** → 탭 이름에 차이를 명시  
예: `2026 신년 가이드` / `2026 명품 토정`
